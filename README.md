# portfolio-dashboard

投資績效儀表板：一個純前端靜態網頁，用來瀏覽你自己的台股／美股投資績效。可依年度、市場篩選，切換顯示幣別並看到匯率換算統計，並透過介面新增／刪除交易紀錄。

## 功能

- 依年度、市場（台股／美股）篩選交易紀錄與績效統計
- 顯示幣別切換（TWD／USD），含即時匯率換算
- 已實現損益、未實現損益、ROI% 總覽卡片與圖表
- 持股現價自動抓取（含來源徽章：即時／快取／估計值／手動覆寫），並可手動覆寫現價
- 新增／刪除交易紀錄，CSV 匯出／匯入備份
- 未匯出異動達門檻時顯示提醒橫幅，離開頁面前若有未匯出異動會跳出瀏覽器確認對話框

## 資料儲存與備份（請務必閱讀）

本應用程式**沒有後端資料庫**，所有交易紀錄、現價快取、手動覆寫、匯率快取、篩選條件都只存在瀏覽器的 `localStorage` 中（key 皆以 `pfd.` 為前綴）。這代表：

- **清除瀏覽器資料、換瀏覽器、換裝置都會遺失所有資料**，且無法復原。
- 請定期使用「匯出 CSV」按鈕備份你的交易紀錄到 `db/tw-stock.csv` / `db/us-stock.csv`（或自訂檔名），並妥善保存。
- 應用程式內建兩個輔助提醒機制，但**僅是輔助、不是保證**：
  - 累計未匯出的新增／刪除異動達 5 筆時，畫面上方會出現提醒橫幅，可點擊「立即匯出」。
  - 嘗試關閉分頁或重新整理時，若有未匯出的異動，瀏覽器會跳出原生的離開確認對話框。

## CSV 欄位 Schema

`db/tw-stock.csv` 與 `db/us-stock.csv`（以及對應的 `.example.csv`）共用以下欄位：

| 欄位 | 說明 |
|---|---|
| `date` | 交易日期，格式 `YYYY-MM-DD` |
| `symbol` | 股票代號（如 `2330` / `AAPL`） |
| `name` | 顯示名稱，可空白 |
| `action` | `buy`（買進）或 `sell`（賣出） |
| `quantity` | 股數，必須大於 0 |
| `price` | 單股成交價；`tw-stock.csv` 內為 TWD，`us-stock.csv` 內為 USD（依檔案隱含幣別，不額外存欄位） |
| `fee` | 手續費，可空白（視為 0） |

匯入 CSV 時，每一列都會被驗證：股數需 `> 0`、單價需 `>= 0`、`action` 必須是 `buy` 或 `sell`、日期格式需正確。不符合規則的列會被略過並回報行號與原因，其餘合法的列仍會正常匯入。**匯入採「附加」而非「取代」**：匯入的資料會加進現有交易紀錄，不會清空原本的資料。

## 如何載入自己的資料

1. 點擊頁面下方「載入範例資料」可先體驗範例資料（`db/tw-stock.example.csv`、`db/us-stock.example.csv`）。
2. 依照上方 CSV Schema 自行建立 `db/tw-stock.csv` / `db/us-stock.csv`（可用 Excel／文字編輯器），或直接使用介面的「新增交易」表單逐筆輸入。
3. 透過頁面上的「匯入 CSV」按鈕，選擇市場後選取你的 CSV 檔案匯入。
4. 之後可隨時用「匯出 CSV」備份目前 `localStorage` 中的最新資料。

### `.gitignore` 行為說明

`db/*.csv` 已被加入 `.gitignore`，避免你的真實個人交易資料被誤提交進版本控制；只有 `db/tw-stock.example.csv` 與 `db/us-stock.example.csv` 兩個範例檔案會被追蹤（用於展示 schema），其餘真實資料檔案不會被 git 追蹤。

## ROI% 計算方式

採用**平均成本法**（非 FIFO）。每次買進都會更新加權平均成本，賣出則用賣出價與當下平均成本的差額計入已實現損益，平均成本本身不因賣出而改變。

```
ROI% = (已實現損益 + 未實現損益) / 總投入成本 × 100
```

其中「總投入成本」是篩選範圍內歷史累計的買入成本，**不會因為賣出而減少**（並非僅目前持股的成本基礎）。這樣設計是為了避免賣掉一部分獲利持股後，ROI% 因分母縮小而被人為放大。當總投入成本為 0 時，畫面會顯示「—」而非 0% 或 NaN。

跨市場合併顯示時，會先用各市場原幣計算完整統計，再用匯率轉換成顯示幣別後加總，最後重新計算一次 ROI%（絕不直接平均兩個市場的百分比）。

## 股價來源免責聲明

持股現價透過專案自帶的 `/api/stock-price` Serverless Function，在伺服器端代為呼叫 Yahoo Finance 的非官方報價端點取得。請注意：

- 此端點為非官方來源，格式或限流規則可能隨時改變或中斷，**重要投資決策請以券商或官方資料來源為準**。
- 台股以代號加上 `.TW` 後綴查詢，**上櫃（TPEx）股票可能因此對應錯誤而抓不到現價**，遇到這種情況請使用「手動覆寫」輸入現價。
- 現價解析順序為：手動覆寫 > 自動抓取的即時報價 > 本地快取的舊報價 > 該標的平均成本（標示為「估計值」）。即使 API 抓取失敗或在純靜態環境下沒有此 API，應用程式仍可正常運作。

## 部署

本專案建議部署在支援 Serverless Functions 的平台：

- **Vercel（建議）**：使用 `vercel` CLI 或連結 GitHub repo 後即可零設定自動部署（免費額度足夠），會同時部署靜態檔案與 `api/stock-price.js`。
- **純靜態檔案託管**（如 GitHub Pages）：頁面仍可正常開啟使用，但因為沒有 `/api/stock-price`，持股現價會自動 fallback 成「估計值」，使用者仍可隨時手動輸入現價。

## 本機開發

- **完整測試（含現價自動抓取）**：安裝 [Vercel CLI](https://vercel.com/docs/cli) 後執行 `npx vercel dev`（第一次需要 `vercel login`），瀏覽器開啟 `http://localhost:3000/`。
- **僅測試非現價功能**：執行 `python3 -m http.server 8000`，瀏覽器開啟 `http://localhost:8000/`。此模式下 `/api/stock-price` 會回傳 404，現價會自動走 fallback 鏈（快取或估計值），其餘功能不受影響。

> 注意：直接用瀏覽器以 `file://` 開啟 `index.html` 會因為 `fetch('db/...')` 等請求被 CORS 擋下而無法正常載入初始資料，請務必透過上述任一種簡易伺服器開啟。

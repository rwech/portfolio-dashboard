# portfolio-dashboard

[![Test](https://github.com/rwech/portfolio-dashboard/actions/workflows/test.yml/badge.svg)](https://github.com/rwech/portfolio-dashboard/actions/workflows/test.yml)

投資績效儀表板：一個純前端靜態網頁，用來瀏覽你自己的台股／美股投資績效。可依年度、市場篩選，切換顯示幣別並看到匯率換算統計，並透過介面新增／刪除交易紀錄。

## 功能

- 依年度、市場（台股／美股）篩選交易紀錄與績效統計
- 顯示幣別切換（TWD／USD），含即時匯率換算
- 總覽頁面提供總投入成本、目前持股成本、總損益（內含已實現／未實現損益子欄位）、ROI% 卡片，以及持股成本配置圖
- 個股損益分頁提供各標的已實現／未實現損益長條圖、持股成本占比圖與明細表
- 持股現價自動抓取（含來源徽章：即時／快取／估計值／手動覆寫），並可手動覆寫現價
- 新增交易：可逐筆填表新增
- 匯入 CSV（交易紀錄分頁）：匯入前先顯示預覽與統計（解析成功／格式錯誤／重複筆數／將新增筆數），確認後才寫入；預設**只新增、自動略過與現有資料重複的紀錄**，重複匯入同一份檔案不會產生重複資料；進階選項可改為「清空該市場後匯入」
- 匯入自動偵測 Big5 編碼（台灣券商匯出檔常見），並在欄位名稱與標準 schema 不同時開啟「欄位對應」步驟：自動依常見券商欄名（成交日期、證券代號、買賣別…）猜測對應，套用後會記住同格式檔案的對應設定；買賣別（買進／賣出）、日期（`2024/1/10`、`20240110`）與千分位數字也會自動正規化
- 交易紀錄：可逐筆刪除（可復原）、搜尋、CSV 匯出備份，內建可展開的「CSV 格式說明」
- 頁首「示範模式」開關：一鍵切換顯示內建範例資料（台股／美股各 60 筆、橫跨 2023～2025 三個年度、各 10 種標的），純記憶體顯示、不會寫入或覆蓋你的真實資料；開啟期間新增、刪除、匯入、價格覆寫、匯出等會修改資料的操作會暫時停用，關閉後立即還原為你自己的真實資料
- 未匯出異動達門檻時顯示提醒橫幅，離開頁面前若有未匯出異動會跳出瀏覽器確認對話框

## 資料儲存與備份（請務必閱讀）

本應用程式**沒有後端資料庫**，所有交易紀錄、現價快取、手動覆寫、匯率快取、篩選條件都只存在瀏覽器的 `localStorage` 中（key 皆以 `pfd.` 為前綴）。這代表：

- **清除瀏覽器資料、換瀏覽器、換裝置都會遺失所有資料**，且無法復原。
- 請定期使用「匯出 CSV」按鈕備份你的交易紀錄到 `db/tw-stock.csv` / `db/us-stock.csv`（或自訂檔名），並妥善保存。
- 應用程式內建兩個輔助提醒機制，但**僅是輔助、不是保證**：
  - 累計未匯出的新增／刪除異動達 5 筆時，畫面上方會出現提醒橫幅，可點擊「立即匯出」。
  - 嘗試關閉分頁或重新整理時，若有未匯出的異動，瀏覽器會跳出原生的離開確認對話框。

## CSV 欄位 Schema

`db/tw-stock.csv` 與 `db/us-stock.csv`（以及對應的 `.example.csv`）共用以下欄位：

| 欄位       | 說明                                                                                         |
| ---------- | -------------------------------------------------------------------------------------------- |
| `date`     | 交易日期，格式 `YYYY-MM-DD`                                                                  |
| `symbol`   | 股票代號（如 `2330` / `AAPL`）                                                               |
| `name`     | 顯示名稱，可空白                                                                             |
| `action`   | `buy`（買進）或 `sell`（賣出）                                                               |
| `quantity` | 股數，必須大於 0                                                                             |
| `price`    | 單股成交價；`tw-stock.csv` 內為 TWD，`us-stock.csv` 內為 USD（依檔案隱含幣別，不額外存欄位） |
| `fee`      | 手續費，可空白（視為 0）                                                                     |

匯入 CSV 時，每一列都會被驗證：股數需 `> 0`、單價需 `>= 0`、`action` 必須是 `buy` 或 `sell`、日期格式需正確。不符合規則的列會被略過，並在匯入預覽中回報行號與原因，其餘合法的列仍可正常匯入。「交易紀錄」分頁有對應的「CSV 格式說明」可展開查看，內容與本節一致。

匯入流程（單一入口，位於「交易紀錄」分頁）：

1. 選擇市場與檔案後，會自動偵測編碼（UTF-8／Big5）。
2. 若欄位名稱與標準 schema 不同（例如券商匯出檔的「成交日期」「證券代號」），會先進入**欄位對應**步驟，自動猜測對應並可手動調整；套用後會記住同格式的對應設定。
3. 接著顯示**匯入預覽**：解析成功筆數、日期範圍、標的數、格式錯誤清單、與現有資料重複的筆數，以及**將新增的筆數**。
4. 預設行為是**只新增、略過重複**（以 日期＋代號＋買賣＋股數＋單價＋手續費 比對），因此定期把券商完整匯出檔直接重匯是安全的；若要整批重來，可勾選進階選項「清空該市場現有紀錄後匯入」，會顯示紅色警告再要求確認。

匯入操作在「示範模式」開啟時會被停用（避免誤觸覆蓋或污染真實資料），請先關閉示範模式再匯入。

## 如何載入自己的資料

1. 勾選頁首的「示範模式」可先體驗內建範例資料（`db/tw-stock.example.csv`、`db/us-stock.example.csv`），此時畫面顯示的是範例資料，**不會**讀取或覆蓋你儲存在 `localStorage` 的真實資料；取消勾選即可立即切回真實資料。
2. 準備 CSV 檔案——可以直接用券商匯出的對帳單（Big5 編碼與常見中文欄名會自動處理），或依照上方 CSV Schema 自行建立；也可以用「新增交易」分頁的表單逐筆輸入（請先確認「示範模式」已關閉）。
3. 到「交易紀錄」分頁點「匯入 CSV」，選擇市場與檔案，依預覽確認後匯入。預設只新增、自動略過重複，不會覆蓋現有資料；若要整批取代，在預覽中勾選「清空該市場現有紀錄後匯入」。
4. 之後可隨時用「匯出 CSV」備份目前 `localStorage` 中的最新資料；下次直接把券商最新的完整匯出檔重匯即可，只會補上新的交易。

### `.gitignore` 行為說明

`db/*.csv` 已被加入 `.gitignore`，避免你的真實個人交易資料被誤提交進版本控制；只有 `db/tw-stock.example.csv` 與 `db/us-stock.example.csv` 兩個範例檔案會被追蹤（用於展示 schema），其餘真實資料檔案不會被 git 追蹤。

## ROI% 計算方式

採用**平均成本法**（非 FIFO）。每次買進都會更新加權平均成本，賣出則用賣出價與當下平均成本的差額計入已實現損益，平均成本本身不因賣出而改變。

```
ROI% = (已實現損益 + 未實現損益) / 總投入成本 × 100
```

其中「總投入成本」是篩選範圍內歷史累計的買入成本，**不會因為賣出而減少**（並非僅目前持股的成本基礎）。這樣設計是為了避免賣掉一部分獲利持股後，ROI% 因分母縮小而被人為放大。當總投入成本為 0 時，畫面會顯示「—」而非 0% 或 NaN。

跨市場合併顯示時，會先用各市場原幣計算完整統計，再用匯率轉換成顯示幣別後加總，最後重新計算一次 ROI%（絕不直接平均兩個市場的百分比）。

## 股價來源免責聲明

持股現價透過專案自帶的 `/api/stock-price` Serverless Function，在伺服器端代為呼叫 Yahoo Finance 的非官方報價端點取得。請注意：

- 此端點為非官方來源，格式或限流規則可能隨時改變或中斷，**重要投資決策請以券商或官方資料來源為準**。
- 台股以代號加上 `.TW` 後綴查詢，**上櫃（TPEx）股票可能因此對應錯誤而抓不到現價**，遇到這種情況請使用「手動覆寫」輸入現價。
- 現價解析順序為：手動覆寫 > 自動抓取的即時報價 > 本地快取的舊報價 > 該標的平均成本（標示為「估計值」）。即使 API 抓取失敗或在純靜態環境下沒有此 API，應用程式仍可正常運作。
